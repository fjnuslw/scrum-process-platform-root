# Scrum 过程管理平台

本项目用于完成《AI 辅助软件过程实验》课程作业，目标是开发一个 Web 版 Scrum 过程管理平台，并完整记录 AI 辅助开发、版本管理、接口测试、问题修复与实验总结过程。

## 1. 项目目标

围绕课程要求，完成以下内容：

- 实现 Scrum 过程管理平台基本版功能
- 使用 AI 工具辅助开发，但由人类负责需求拆解、结果校验和错误修正
- 使用 Git 记录完整开发过程
- 编写并执行接口自动化测试
- 保留 AI 交互记录、问题修复记录、版本截图和实验报告材料

## 2. 模式选择

本项目采用 **Builder 模式思路** 进行个人主导开发：

- 前期可保留少量讨论记录以满足课程要求
- 实际开发、代码仓库、AI 会话记录、测试脚本与总结材料全部独立维护
- 不采用 Solo + OpenSpec 路线，避免额外的规范驱动协作成本

## 3. 功能范围

仅优先完成课程要求中的**基本版功能**：

1. 用户故事管理
2. 产品待办列表管理
3. 迭代计划管理
4. 迭代开发管理（看板 + 燃尽图）
5. 迭代回顾管理

不主动开发增强版，除非基本版、测试和报告都已完成。

## 4. 推荐技术栈

- 前端：Vue 3 + Vite + Element Plus
- 后端：Node.js + Express
- 数据库：MySQL
- 接口测试：Postman/Newman 或 supertest
- 图表：ECharts
- 拖拽：SortableJS 或 vue-draggable-plus

## 5. 根目录说明

- `README.md`：项目总说明与启动指引
- `AGENTS.md`：给 Codex 的项目级行为约束
- `.gitignore`：Git 忽略配置
- `docs/项目执行与技术说明.md`：完整执行方案与交付要求解释
- `docs/数据库设计.md`：数据库表结构建议
- `docs/API接口草案.md`：核心 REST 接口设计
- `docs/开发任务清单.md`：开发顺序与完成标准
- `prompts/codex-开场提示词.md`：给 Codex 的首轮提示词模板
- `prompts/trae-builder-开场提示词.md`：给 Trae Builder 的首轮提示词模板

## 6. 开发建议顺序

1. 初始化 Git 仓库与项目目录
2. 建立数据库与后端基础接口
3. 完成用户故事管理
4. 完成待办列表排序与筛选
5. 完成迭代管理与故事加入迭代
6. 完成看板状态流转
7. 实现燃尽图
8. 实现回顾管理
9. 编写接口自动化测试
10. 整理报告与录屏材料

## 7. 本地使用 AI 的建议方式

### Codex

适合做：

- 阅读仓库并理解已有结构
- 检查接口与数据模型一致性
- 补测试
- 修复 bug
- 做小步迭代改动

### Trae IDE / Builder

适合做：

- 生成页面骨架
- 快速搭 CRUD
- 根据明确需求生成模块代码
- 做局部 UI 与交互补全

## 8. 注意事项

- AI 每次只处理一个小任务，不要一次要求改完整个项目
- 每次 AI 输出后必须人工验证
- 必须记录至少 2 个 AI 生成错误及修正过程
- Git 提交信息要具体清晰
- 报告重点不只是功能，而是过程、测试、反思与协作痕迹
