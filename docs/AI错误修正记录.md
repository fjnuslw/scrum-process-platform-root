# AI 错误修正记录

本文档用于记录 Builder 模式下 AI 生成内容中的实际错误、人工发现方式和修正结果，便于课程报告引用。

## 案例 1：燃尽图测试断言时间点错误

- 指令内容：补充“看板状态更新会驱动燃尽图数据”的后端自动化测试。
- AI 输出结果：初版测试直接断言 `actual[0].remaining === 0`。
- 问题现象：测试失败，返回值为 `8`，而不是预期的 `0`。
- 人工发现方式：运行 `npm test` 后查看 Vitest 失败信息，发现断言拿的是燃尽图第一个日期点，而不是状态变更当天的日期点。
- 修正方法：将断言改为查找“当天日期”对应的实际线数据，再判断 `remaining === 0`。
- 修正后效果：接口测试全部通过，能够正确证明“状态更新后，燃尽图数据会被同步更新”。

## 案例 2：初版后端出现文档外调试接口

- 指令内容：实现 stories 与 backlog 相关接口。
- AI 输出结果：初版在 `stories` 路由里额外生成了一个内部调试接口 `GET /api/stories/_backlog/internal`。
- 问题现象：该接口不在 `docs/API接口草案.md` 和 `docs/项目执行与技术说明.md` 约定范围内，容易造成前后端接口清单不一致。
- 人工发现方式：在整理接口列表和对照课程文档时发现该路径属于多余设计。
- 修正方法：删除该内部调试接口，仅保留文档中定义的正式接口。
- 修正后效果：后端接口集合与课程文档保持一致，便于后续测试、截图和报告撰写。
