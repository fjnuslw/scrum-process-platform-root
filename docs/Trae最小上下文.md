# Trae 最小上下文

## 1. 当前项目状态

- 仓库已完成基础版前后端框架。
- 后端已实现核心 REST 接口与自动化测试。
- 前端已实现用户故事、backlog、迭代、看板、燃尽图、回顾页面。
- README、阶段截图、AI 错误修正记录、报告底稿已整理。
- 当前工作区干净，可继续增量开发。

## 2. 关于工具使用的真实写法

- 前期先结合 Codex 完成文档收敛、原型搭建、接口和测试校对。
- 后续正式实验阶段转入 Trae IDE Builder 模式继续做增量开发与过程留痕。
- 不伪造“全部代码都由 Trae 从零生成”。

## 3. 当前最重要目标

补 Trae Builder 证据链，并完成真实的增量开发任务。

## 4. 推荐的第一个 Trae 任务

把“故事加入迭代”从当前选择式交互改为更接近课程要求的拖拽式交互。

原因：

- 任务边界清晰
- 改动范围可控
- 容易形成完整的 Builder 证据链
- 能真实增强项目对课程要求的贴合度

## 5. 后续优先级

1. 故事加入迭代改为拖拽式
2. 后端持久化切换到 MySQL
3. 补 1~2 个 Bug 修复闭环
4. 补测试材料
5. 补 Gitee 远程仓库证据

## 6. 运行说明

- 后端端口：`3000`
- 前端端口：`5173`
- 若再次运行 `npm run dev` 报端口占用，先检查是否已有残留 `node` 进程在监听端口。
